# Obsidian And Orst Onboarding Primer

Use this primer before onboarding work.

## Obsidian Basics

- Use wiki links (`[[Note]]`) and explicit project paths for clarity.
- Keep note frontmatter complete for metadata and graph queries.
- Use overview notes and direct links.

## Orst Model

- Start at [[Vault Home]].
- Shared OS sits at root (`coding-guidelines.md`, `templates/`, `skills/`, `playbooks/`, `automations/`).
- Project truth lives in `projects/<project>/`.

## Project Intake Sequence

1. `projects/<project>/Project Overview.md`
2. `projects/<project>/status/Current State.md`
3. `projects/<project>/knowledge/`
4. `projects/<project>/workflows/Agent Onboarding.md` (if present)

## Boundary Rule

- Default write scope: assigned project + shared root OS.
- Cross-project edits require explicit user approval.
